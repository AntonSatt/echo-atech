# atech

Firmware projects for [Atech](https://atech.dev) 14-port boards (ESP32-S3), built with the
`atech` SDK. Started as module experiments, grew into a two-game arcade, a multi-board
ESP-NOW light show, and hackathon material for the MagasinX hardware hackathon (July 2026).

## Projects

| Directory | What it is |
|---|---|
| `smoke_test/` | Minimal sanity check — prints a heartbeat event over serial once per second. Flash this first to verify the toolchain and board. |
| `calibrated/` | Two-game precision arcade on a single board. **Blind Spin**: turn the knob an exact number of degrees with the screen dark. **Sixth Sense**: count a time interval in your head, then wave at the PIR sensor. Three rounds each, scored out of 300, with best-score tracking. |
| `conductor/` + `dancer/` | Multi-board synced light show over ESP-NOW (no router needed). The conductor's rotary knob picks color (turn) and pattern (press) and broadcasts `{pattern, hue, clock}` at 10 Hz; every dancer renders the same animation phase-locked to the conductor's clock. Dancers fall back to a dim blue "searching" breathe when no conductor is heard for 3 s. |
| `light_modules/` | Custom Atech modules shared via `modules_path`. Currently `espnow_link/` — the board-to-board broadcast link used by conductor/dancer, including the shared pattern renderer and HSV helpers. |
| `inspectorq/` | Hackathon plan docs for **InspectorQ**, a €60 VLM-powered visual quality-inspection cell (UNO Q + camera + Atech stack-light). Plans only, no code — the team pivoted on day 2, kept here in case it's revisited. |
| `nvs_config.csv` | *(generated, gitignored — contains WiFi credentials)* NVS provisioning table for a RuView WiFi-CSI sensor node (day-2 hackathon build), regenerated by RuView's `provision.py` on each run: AP credentials, target host/port, node id, channel. |

## Setup

The SDK lives in a local virtualenv (PlatformIO Core is pulled in as its build backend —
no IDE extension needed):

```bash
python3 -m venv .venv
source .venv/bin/activate   # atech needs the venv on PATH to find PlatformIO
pip install atech           # this repo was built against 1.0.0a3
```

The board enumerates as `/dev/ttyACM*`. If you're not in the `dialout` group, you'll need
`sudo chmod 666 /dev/ttyACM0` after every replug (or fix it once with
`sudo usermod -aG dialout $USER` and re-login).

## Workflow

```bash
atech validate <project_dir>
atech build <project_dir>
atech upload <project_dir>
```

Each project is a `project.yaml` declaring the board, the modules with their port
assignments, and inline `code.setup` / `code.loop` blocks. The SDK generates the
PlatformIO project into `<project_dir>/build/` (gitignored).

### Hardware per project

- **calibrated** — button (port 1), TFT (3+4), rotary encoder (5+6), PIR (7), NeoPixel grid (9), speaker (13+14)
- **conductor** — espnow_link (2, uses no pins), TFT (3+4), rotary encoder (5+6), NeoPixel grid (9)
- **dancer** — espnow_link (1), NeoPixel grid (7). Flash to as many boards as you have.

## SDK notes (hard-won)

- Generated module loop templates run **before** user `code.loop` and consume all `was*()`
  edge methods — edge-detect yourself from level reads (`isPressed()`, `isMotionDetected()`).
- Only `code.setup`/`code.loop` exist in `project.yaml`; anything needing file-scope code
  (includes, ISR/callbacks, classes) must be a custom module via `modules_path` — see
  `light_modules/espnow_link/` for the pattern.
- Avoid `link` as a module instance name — it collides with POSIX `link()` in the
  generated `main.cpp`.
- Rotary encoder: 18 detents/rev (20°/detent); call `setAcceleration(false)` for 1 count
  per detent, and `enableRing(false)` if the LED position ring would give the game away.
- PIR (AM312) is motion yes/no only: ~30 s warmup, ~2 s re-arm between triggers.
- ESP-NOW here runs router-free on channel 1. If a board ever needs ESP-NOW *and* real
  WiFi, `espnow_link` must move to the router's channel — one radio, one channel.

## Known issues

- `calibrated/` Blind Spin scores 0 on every round — suspected `getPosition()` returns
  more than 1 count per detent (so 360° reads as 720°). Not yet verified; needs a raw
  encoder-event test before trusting the scoring math.
